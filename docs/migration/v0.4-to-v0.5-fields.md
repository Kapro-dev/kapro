# v0.4 to v0.5 Field Migration

Kapro v0.5.9 introduces backend-neutral delivery names while keeping the old
fields as compatibility aliases.

## Backend

Old:

```yaml
spec:
  driver: argo
  adapter: argo-cd
  runtime: Hub
```

New:

```yaml
spec:
  substrate:
    kind: argo
    actuator: argo
  execution:
    mode: hub-push
```

Mapping:

| Old | New |
| --- | --- |
| `driver: argo` | `substrate.kind: argo` |
| `driver: flux` | `substrate.kind: flux` |
| `driver: oci` | `substrate.kind: oci` |
| `adapter: argo-cd` | `substrate.actuator: argo` |
| `runtime: Hub` | `execution.mode: hub-push` |
| `runtime: Spoke` | `execution.mode: spoke-pull` |

Do not set both shapes on the same object in v0.5. In v0.6 the old fields are
removed from the public CRD; use the new shape before applying 0.6 manifests.

Run:

```bash
kapro doctor
```

For the 0.6 reset, follow [Pre-0.6 API Reset](pre-0.6-api-reset.md).

## Cluster Provider Kind

`Cluster.spec.provider.kind` now uses an open DNS-style string instead of a
closed enum. Existing values such as `outbound-agent`, `gcp-fleet`,
`gcp-connect-gateway`, `eks`, `aks-arc`, `rhacm`, `capi`, and `kubeconfig`
remain valid.
